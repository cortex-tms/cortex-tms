## Synopsis

```bash
cortex-tms dashboard [options]
```

## Description

The dashboard is built with **React + Ink** â€” it runs entirely in your terminal, no browser required. Navigate between three views using the `1`, `2`, `3` keys and quit with `q` or `Ctrl+C`.

<Aside type="tip" title="Live Mode">
Run `cortex-tms dashboard --live` to auto-refresh every 5 seconds â€” ideal for keeping a governance health pane open while you work.
</Aside>

---

## Options

### `--live`

Enable auto-refresh. The dashboard re-runs `collectTMSStats()` every 5 seconds and updates all cards.

```bash
cortex-tms dashboard --live
```

### `--cwd <path>`

Run the dashboard against a different project directory instead of the current working directory.

```bash
cortex-tms dashboard --cwd /path/to/project
```

## How It Works

The dashboard calls the same `collectTMSStats()` pipeline used by `cortex-tms validate` and `cortex-tms status`. All data is read from:

- `.cortexrc` â€” project configuration
- `NEXT-TASKS.md` â€” sprint detection
- Git history â€” staleness detection (requires `fetch-depth: 0` in CI)
- Last validation/review results â€” validation and Guardian cards

No external API calls are made. All data is local.
