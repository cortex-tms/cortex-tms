## Completed Features

### 1. Git Hook Integration — `cortex-tms hooks`

**Commit**: `ba39cf0`
**Branch**: `feat/hooks-install`

Adds a pre-commit hook that runs `cortex-tms validate` before every commit,
preventing context rot from reaching the repo.

**What shipped**:
- `cortex-tms hooks install` — installs pre-commit hook
- `cortex-tms hooks uninstall` — removes hook safely
- `cortex-tms hooks status` — shows current hook configuration
- `--strict` and `--skip-staleness` flags for hook mode
- Binary resolution: local `node_modules/.bin` → global → `npx` fallback
- Foreign hook detection (refuses to overwrite hooks it didn't create)
- 16 tests

**Config additions**: `hooks?` field in `.cortexrc`, `skipStaleness?` in `ValidateCommandOptions`

### 2. AGENTS.md Template — Multi-Agent Governance

**Commit**: `9f895b8`
**Branch**: `feat/agents-md-template`
**Plan**: `docs/plans/agents-md-plan.md`

Adds `AGENTS.md` as a tool-agnostic governance source-of-truth for projects
using multiple AI agents (Claude Code, Cursor, Copilot, Windsurf, OpenCode, etc.).

**What shipped**:
- `templates/AGENTS.md` — Precedence, Active Agents & Roles, Capabilities Matrix,
  Handoff Protocol, Escalation Rules, Universal Prohibitions
- Included in `standard` and `enterprise` scope presets (optional)
- Line limit: 300 lines
- `validateRecommendedFiles()` — emits `passed:true, level:"info"` hint when
  AGENTS.md is missing (standard/enterprise scope only; nano is excluded)
- `💡 Recommendations` display section in `validate` output
- `validate --strict` exits 0 when only AGENTS.md is missing
- 14 tests

**Key design decision**: `passed:true, level:"info"` ensures missing AGENTS.md
never reduces the "Passed: X" summary count or fails strict mode.

### 3. Governance Packs — `cortex-tms init --preset node`

**Branch**: `feat/governance-packs-node`

Adds `--preset <preset>` flag to `init`. Preset controls ecosystem-specific
template content independently of `--scope` (which controls the file set).
The Node.js pack is the first shipped preset.

**What shipped**:
- `--preset node` flag — composable with any scope
- Six Node-specific template overrides:
  - `docs/core/PATTERNS.md` — async/await, error handling, module structure, test patterns
  - `docs/core/DOMAIN-LOGIC.md` — Node idioms, event loop, streaming, package manager conventions
  - `docs/core/ARCHITECTURE.md` — common Node layouts, dependency conventions
  - `CLAUDE.md` — agent instructions with Node stack section
  - `AGENTS.md` — multi-agent governance with Node-specific prohibitions
  - `.github/copilot-instructions.md` — agent rules with Node stack, no generic hints
- Preset resolution: preset override → base template fallback per file
- `metadata.preset` recorded in `.cortexrc`
- `templates/presets/` excluded from base walker — no path collision
- 17 tests: unit (resolver, walker, schema) + E2E (both `--minimal` and `--scope nano`
  code paths, `--dry-run`, `.cortexrc`, invalid preset rejection, placeholder safety)
- README and website CLI reference updated

**Key design decisions**:
- `preset` stored in `metadata` (not top-level) — init provenance only for now;
  promote to top-level if `migrate`/`validate`/`status` need preset-awareness
- No bracket-style placeholders in preset templates — HTML comments inline for
  user-editable values, avoiding collision with the three known template placeholders
- Python/Go packs deferred — validate Node with real users first
