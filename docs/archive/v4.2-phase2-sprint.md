### TMS-423 — Prevent live Guardian accuracy tests from running in normal verify flows

**Closed**: 2026-05-12
**Commits**: `1c03867` (implement), `de21cc3` (push)
**Repo**: cortex-tms

Root cause of TASK-002 verify timeout: `guardian-accuracy.test.ts` makes
24 live Anthropic API calls when `ANTHROPIC_API_KEY` is present — not parallel
temp-dir contention as initially diagnosed (`mkdtemp()` is atomic, no collision
possible).

**What shipped**:
- `CORTEX_SKIP_LLM_TESTS=1` guard at the top of `beforeAll`, before the
  `ANTHROPIC_API_KEY` check (guard order is critical — key check must come second)
- `max_verify_retries` lowered from 1 → 0 in ai-planner cortex-tms project config
- TMS-421 findings doc corrected to remove the "parallel temp-dir contention"
  misdiagnosis and document the real root cause

**Correction**: Earlier diagnosis in TMS-421 findings was wrong. Corrected in
both the findings doc and NEXT-TASKS.md closure text.

### TMS-424 — Per-project `verify_env` support in ai-planner

**Closed**: 2026-05-12
**Commits**: `f2d9f5c` (implement), `1135da6` (merge)
**Repo**: ai-planner

Generalises the credential-strip fix. Some tests need explicit env opt-outs
rather than blanket scrubbing.

**What shipped**:
- `project.yaml` accepts optional `verify_env: { KEY: "value" }` block
- `step_8_verify` merges `verify_env` into the scrubbed subprocess env after
  credential stripping (order: strip credentials first, then apply project overrides)
- Schema validation rejects non-string values
- cortex-tms project config updated: `CORTEX_SKIP_LLM_TESTS: "1"`

**Key design decision**: credential strip runs first, then project overrides.
A malicious `verify_env: { ANTHROPIC_API_KEY: "..." }` cannot re-inject
stripped keys because the scrub happens unconditionally before the merge.

## Lessons

- **Verify/review gap is real**: TASK-001 and TASK-004 both passed verify
  with wrong intent. Review entries in `runs.jsonl` close this gap structurally.
- **Misdiagnosis compounds**: The "parallel temp-dir contention" diagnosis was
  wrong and propagated to 3 documents before being corrected. Always check
  the simplest explanation first (`mkdtemp()` is atomic by design).
- **Docs rot silently**: `auto-tier` had 7 phantom flags in CLI-USAGE.md that
  passed `validate --strict` because validation checks structure, not accuracy.
  Human (or external model) review is required for semantic correctness.
