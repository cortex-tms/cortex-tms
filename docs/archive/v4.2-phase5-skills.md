## Context

Phase 5 closes the v4.2 sprint. Phases 1-3 shipped governance preset content; Phase 4 shipped
the MCP server; Phase 5 ships the Claude Code skill layer — the third surface through which
TMS governance docs reach AI agents at runtime.

The design went through four reviewer passes (GPT-5.5, Kimi K2.6 × 2, Claude Opus 4.7) and
official-doc verification before any code was written. The key locked decisions:

- **Distribution**: bundled in the npm package, opt-in via `--with-skills` (Q1)
- **Doc access**: direct file reads from the user's project, no MCP coupling (Q2)
- **Install path**: `.claude/skills/cortex-<name>/SKILL.md` — directory name = slash command (Q3)
- **Prefix**: `cortex-` prefix on all shipped skills to prevent name collisions (Q4/Kimi)
- **Default**: off — non-Claude-Code users see no change to `init` (Q5)
- **Versioning**: manual re-run for updates in v1 (Q6)
